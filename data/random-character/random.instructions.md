# Random Character Data - AI Instructions

## 폴더 목적

이 폴더는 위빙 플랫폼의 랜덤 캐릭터 생성 기능을 위한 데이터를 관리합니다.
본 폴더는 push 시 자동으로 서버에서 데이터가 갱신되어야 합니다.

## 데이터 구조 규칙

### JSON 파일 구조

모든 JSON 파일은 다음 구조를 따라야 합니다:

```json
{
  "traits": {
    "category_name": ["항목1", "항목2", "항목3"],
    "another_category": ["다른항목1", "다른항목2"]
  }
}
```

### 카테고리 명명 규칙

- `snake_case` 사용 (예: `body_type`, `hair_color`)
- 영어 카테고리명 사용하되 한국어 설명 추가 가능
- 일관성 있는 명명 유지

### 데이터 작성 가이드라인

#### 1. 한국어 콘텐츠

- 모든 특성 항목은 한국어로 작성
- 자연스러운 한국어 표현 사용
- 어색한 번역체 지양

#### 2. 장르별 특화

- `common.json`: 모든 장르 공통 특성
- `fantasy-east.json`: 동양 판타지 (무협, 선협)
- `fantasy-west.json`: 서양 판타지
- `sf.json`: 공상과학
- `horror.json`: 호러/스릴러
- `reality.json`: 현대/현실

#### 3. 콘텐츠 품질

- 창의적이고 다양한 특성 제공
- 클리셰 지양, 독창적 요소 포함
- 캐릭터의 개성을 살릴 수 있는 구체적 특성

#### 4. 데이터 일관성

- 동일한 카테고리는 모든 파일에서 일관된 형태 유지
- 항목 수는 최소 10개 이상 권장
- 중복 항목 제거

## 파일 수정 시 주의사항

1. **JSON 문법 검증**: 수정 후 반드시 JSON 유효성 검사
2. **인코딩**: UTF-8 인코딩 유지
3. **백업**: 대량 수정 전 백업 생성
4. **테스트**: 랜덤 생성 기능 테스트로 검증

## 새 장르 추가 시

1. 기존 `common.json`의 카테고리 구조 참고
2. 장르별 특화 요소 추가
3. 파일명: `{장르명}.json` 형식
4. 최소 5개 이상의 카테고리 포함

## 관련 코드 연동

이 데이터는 `/hooks/use-random-character.ts`에서 사용되므로, 구조 변경 시 해당 파일도 함께 확인하세요.
